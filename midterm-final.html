Requirements of Midterm

  • Final submission of at least 3 wireframes of website in desktop and mobile sizes based on project brief and updated based on class feedback. Wireframes can be made using Whimsical wireframes or another tool and should convey the key elements of the website's design and layout. The mobile wireframes should show how the layout will adapt to a mobile width. See slides at the end of Class 2 for more tips
  • Final submission of sitemap of website, updated from class feedback. The site map can also be made in Whimsical Wireframes, Slick plan, or another tool. The site map should convey your site's structure. Please map out the entire intended structure even if we will only be able to design the first few pages in this course. See for more tips
  • A style guide for your website design. Style guide should have:
    • Type specs for h1-h4 (or more)
    • body text
    • link text including hover style
    • colors
    • button styles
    • form styles
    • any icons used
    • any other UI elements that are used repeatedly
  • At least 3 pages of website design in desktop and mobile sizes, built in Figma
  • On the day of the midterm, students will present their style guides and website design (final wireframes, sitemap, and project brief should be submitted but not presented). Explain your design decisions as they relate back to your project brief and principles of good design. Explain the feedback you are received so far and how you incorporated or considered it in your design. Let the audience know what areas you are most looking for feedback on. Provide feedback to other presenters
  • See slides and additional reading from class 1 and class 2 for more tips on sitemaps, wireframes, and style guides

Requirements of Final

  • A website with at least 3 pages built with HTML and CSS (bonus points for more)
  • Should be on Github
  • Should include at least 1 feature with Javascript functionality
  • Should be responsive to different screen sizes
  • Be able to explain how website is built with principles of accessibility
  • Be able to explain how you used your code to achieve different styles
  • Be able to connect the design back to the project brief in terms of goals, audience, etc.
  • Identify areas you would like to improve, get feedback on, and keep working on
  • Provide feedback to others.
